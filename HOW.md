Claudeから返答
```
Claude Codeのデモ用に、5〜10分で動かせることを優先したSPEC.mdを作成します。デモを5〜10分に収めるために、特に以下の点で「割り切り」を入れています。

- **DnDは対象外** — カンバンらしさはカラム表示で出し、ステータス変更はセレクトボックス。`@dnd-kit/core` は時間が余ったら、という位置づけにしています
- **編集機能なし** — 削除→再作成で代替（モーダルが1種類で済む）
- **Server Actions のみ** — API Route を作らないので、ファイル数とボイラープレートが減ります
- **DBコンテナを立てない** — SQLiteのファイルマウントだけにして `docker-compose.yml` を1サービスに
- **シードデータ3件** — 起動直後に「動いている」ことが視覚的にわかる

デモ当日に詰まりやすいポイントとしては、Next.js 16のApp Routerで `cookies()` や `headers()` が非同期APIになっている点、Prismaの `output` パス（`@prisma/client` の生成先）が最近のバージョンで明示必須になっているケースがある点あたりです。リハーサルで一度通しておくと安心だと思います。

このSPECで進めて何か違和感があれば、調整します。
```
